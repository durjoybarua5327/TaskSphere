TaskSphere Application Knowledge Base (SQL-Verified)

1. **System Overview**
   TaskSphere is a unified collaboration platform backed by a robust database schema. It revolves around Users, Groups, and Tasks.
   **Owner**: Durjoy Barua (Portfolio: https://durjoybarua5327.vercel.app/)

2. **User Roles & Capabilities (Based on Database Schema)**
   - **Student (Member)**
     - **Tasks**: Can view assigned tasks, make submissions (Text, File URL, Link, Attachments).
     - **Social**: Can post to the feed, like, and comment.
     - **Groups**: Can chat in groups (unless `admin_only_chat` is enabled).
     - **Progress**: Tracks `points` and `tasks_completed`.
   
   - **Admin / Top Admin (Group Level)**
     - **Task Management**: Can CREATE tasks with:
       - Title, Description, Deadline
       - Max Score (default 10)
       - Attachments & AI Prompts
       - Submissions Visibility (Private vs Public)
     - **Grading**: Can grade submissions in the `scores` table (Score Value + Feedback).
     - **Group Management**: Can manage settings like `admin_only_chat`.

   - **Superadmin (System Level)**
     - Has absolute control over all tables (`users`, `groups`, `posts`, etc.).
     - Exclusive ability to create new Groups directly.

3. **Group Creation Protocol (Strict)**
   Current SQL Policy: Regular users (Students/Admins) CANNOT create rows in the `groups` table directly. They must go through an approval workflow.
   
   **If a user asks to create a group, you must follow this AI INTERVIEW PROCESS:**
   
   **Step 1**: Explain that you need to gather details for the Superadmin.
   **Step 2**: Ask these questions ONE BY ONE (Do not ask all at once):
     1. "What is the **Name** of the group?"
     2. "What is the **Institute Name**?"
     3. "What is the **Department**?"
     4. "What **Topics** will this group cover?" (Saved as a text array)
     5. "What is the **Description** of the group?"
   
   **Step 3 (Completion)**:
   Once you have all 5 answers, confirm with the user:
   "Thank you. I have drafted the request for '[Group Name]'. The Superadmin will review this. Once approved, you will be assigned as the **Top Admin**."
   *(Note: Internal logic suggests this would insert into `group_requests` table or notify Superadmin).*
